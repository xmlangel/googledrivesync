# 개발 및 빌드 검증 지침

코드 수정 시 안정성을 보장하기 위해 다음 절차를 엄격히 준수합니다.

## 1. 계획 및 분석 (Planning)

- 수정하려는 코드의 로직과 영향 범위를 분석합니다.
- `implementation_plan.md`를 작성하여 수정 방향을 정의하고 사용자 검토를 받습니다.

## 2. 코드 구현 (Implementation)

- 계획된 설계에 따라 코드를 수정하거나 새로운 기능을 추가합니다.
- 코틀린 코딩 컨벤션 및 프로젝트의 아키텍처 원칙을 준수합니다.

## 3. 빌드 검증 (Build Verification)

수정이 완료된 후, 반드시 다음 명령어를 실행하여 컴파일 오류가 없는지 확인합니다.

// turbo

```bash
./gradlew assembleDebug
```

> [!IMPORTANT]
> 빌드에 실패한 경우, 오류 메시지를 분석하여 즉시 수정하고 다시 빌드를 시도해야 합니다. 빌드가 성공하지 않은 코드는 제출하지 않습니다.

## 4. 단위 테스트 실행 (Testing)

기존 기능의 회귀 오류를 방지하기 위해 테스트를 실행합니다.

// turbo

```bash
./gradlew test
```

## 5. 결과 보고 (Final Walkthrough)

- 모든 수정 사항과 빌드/테스트 결과를 `walkthrough.md`에 요약하여 제출합니다.
- UI 변경이 있는 경우 스크린샷이나 녹화본을 첨부합니다.
