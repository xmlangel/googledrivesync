# 릴리즈 노트 v1.5.0

Google Drive Sync v1.5.0 버전이 릴리즈되었습니다. 이번 버전은 v1.4.0 이후 진행된 여러 마이너 업데이트(v1.4.1 ~ v1.4.4)의 성과를 하나로 통합하고, 동기화 엔진의 안정성과 신뢰성을 극대화하는 데 초점을 맞추었습니다.

## 주요 개선 사항 (v1.4.0 이후 통합)

### 1. 더티 트래킹(Dirty Tracking) 및 실시간 감지
- 로컬 파일 시스템의 변경 사항을 실시간으로 감지하는 `Dirty Tracking` 아키텍처를 도입했습니다.
- 전체 스캔 없이 변경된 파일만 타겟팅하여 동기화하므로 배터리 소모와 API 호출을 획기적으로 줄였습니다.

### 2. 동기화 탄력성(Resilience) 강화
- 특정 파일 하나에서 오류가 발생하더라도 전체 동기화 프로세스가 중단되지 않고 다음 파일로 진행되도록 개선했습니다.
- 개별 파일 단위의 상세한 오류 로그를 제공하여 문제 해결이 용이해졌습니다.

### 3. 코루틴 및 취소 로직 안정화
- 동기화 도중 앱을 종료하거나 작업을 취소할 때 발생할 수 있는 비정상 종료를 방지하기 위해 코루틴 취소 로직을 정교화했습니다.
- `CancellationException`을 올바르게 전파하여 자원 해제가 안전하게 이루어집니다.

### 4. 고급 충돌 해결 전략 구현
- **둘 다 유지(Keep Both)**: 충돌 발생 시 로컬 버전을 복사본으로 남기고 서버 버전을 다운로드하여 데이터 유실을 0%로 유지합니다.
- **매번 확인(Manual Resolution)**: 정책이 설정되지 않은 경우 사용자가 직접 결정할 때까지 파일을 보호합니다.
- 동기화 엔진의 모든 충돌 시나리오에 대한 포괄적인 단위 테스트를 완료했습니다.

### 5. 가용성 및 성능 최적화
- **네트워크 진단**: DNS 오류나 서버 응답 지연 등 네트워크 문제를 더 명확하게 진단하고 로그에 기록합니다.
- **MD5 기반 스킵**: 파일 내용이 사실상 동일한 경우 타임스탬프 차이를 무시하고 전송을 스킵하여 효율을 높였습니다.

## 기술적 성과
- 동기화 핵심 엔진(`SyncManager`, `SyncWorker`)에 대한 테스트 커버리지 대폭 확대.
- 안드로이드 백그라운드 제약 환경(`WorkManager`)에서의 동작 안정성 검증 완료.
