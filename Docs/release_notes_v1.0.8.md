# 릴리즈 노트 v1.0.8

v1.0.8 버전에서는 특정 상황에서 발생하는 중복 동기화 문제를 완벽히 해결하기 위한 '기존 파일 연결(Linking)' 기술을 도입했습니다.

## 주요 변경 사항

### 기존 파일 연결(Linking) 로직 도입

- **데이터 전송 없는 매칭:** 로컬과 구글 드라이브 양쪽에 이미 파일이 존재하고 크기 및 시간이 거의 같다면(2초 이내), 데이터를 전송하지 않고 데이터베이스에 즉시 동기화된 것으로 등록(Match)합니다.
- **초기 동기화 최적화:** 800여 개의 파일이 있는 경우에도 초기 1회만 스캔하면 이후부터는 완벽히 동기화 상태가 유지됩니다.

### 동기화 정합성 및 안정성 보강

- **크기 기반 변경 감지:** 수정 시간뿐만 아니라 파일의 크기(Size) 변화도 실시간으로 감지하여 동기화 정확도를 높였습니다.
- **데이터베이스 정합성 수정:** 드라이브 파일의 정확한 크기가 데이터베이스에 저장되지 않던 문제를 수정했습니다.

## 기술적 개선

- `SyncManager.kt`의 비교 임계값을 1초에서 2초로 상향 조정하여 다양한 파일 시스템 간의 호환성을 높였습니다.
- 기존 파일 매칭 시나리오에 대한 단위 테스트는 다음 버전에 보강될 예정입니다.
